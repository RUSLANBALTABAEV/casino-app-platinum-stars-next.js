## Веб-админ панель

Мини-приложение дополнилось разделом `/admin`, где можно отслеживать метрики и управлять пользователями, промокодами, заданиями и транзакциями.

### 1. Секрет администратора

Добавьте переменную в `.env.local`:

```
ADMIN_SECRET=your-strong-secret
```

Без этой переменной доступ к `/admin` не ограничивается (панель открыта). При наличии секрета пользователю нужно ввести его на странице `/admin/login`. После удачной валидации создаётся httpOnly-cookie, и админ попадает в панель.

### 2. Разрешения middleware

Файл `middleware.ts` перехватывает запросы к `/admin/**` и проверяет cookie `admin_session`. Для смены секрета очистите cookie (кнопка «Выйти» в панели или ручное удаление).

### 3. Разделы панели

| Раздел            | Файл                                           | Описание                                                                    |
|-------------------|------------------------------------------------|------------------------------------------------------------------------------|
| Обзор             | `app/admin/(dashboard)/page.tsx`               | Метрики: пользователи, задания, промокоды, суммы звёзд, последние транзакции. |
| Пользователи      | `app/admin/(dashboard)/users/page.tsx`         | Просмотр профилей, изменение баланса, блокировка/разблокировка.             |
| Промокоды         | `app/admin/(dashboard)/promo/page.tsx`         | Список промокодов, включение/выключение, форма создания новых кодов.        |
| Задания           | `app/admin/(dashboard)/tasks/page.tsx`         | Список активностей спонсоров и форма создания новых заданий.                |
| Игры              | `app/admin/(dashboard)/games/page.tsx`         | Настройка рулетки (колесо/слот), кейсов, лотерей и раннера.                 |
| Транзакции        | `app/admin/(dashboard)/transactions/page.tsx`  | Журнал платежей с фильтрами по статусу и провайдеру.                         |

Все формы реализованы серверными actions и сразу обновляют данные/метрики. Для корректной работы требуется применённая миграция Prisma (см. `docs/database.md`).

### 4. Изменения в схеме БД

Модель `User` получила поле `isBanned`, которое отображается и управляется из панели. Не забудьте выполнить:

```
npx prisma migrate dev --name add_admin_fields
```

### 5. Известные ограничения

- Панель опирается на один секрет и cookie. При необходимости расширенной RBAC добавьте роли в схемe.
- Транзакции отображаются только последние 50 записей; при необходимости добавьте пагинацию.
- Серверные actions не выводят сообщения об ошибках на клиенте — при ошибке произойдёт redirect с исключением в логах.

### Режим заглушки

Если `DATABASE_URL` не задан, админка автоматически переходит в демонстрационный режим: показывает статические примеры данных и отключает формы. Это позволяет просмотреть UI без реальной базы. Чтобы включить полноценную работу, настройте подключение к Postgres и выполните миграции.
