# Referral Program

## User Identity

- Каждый профиль синхронизируется с Telegram: в базе хранится оригинальный `username` без `@`. Система запрещает дубликаты, поэтому имя всегда уникально.
- Для каждого пользователя автоматически создаётся реферальный код (`user.referralCode`), доступный через `GET /api/mini-app/profile` или `GET /api/mini-app/referral`.

## Привязка реферала

1. Клиент мини-приложения отправляет `POST /api/mini-app/referral` с телом `{ "action": "attach", "code": "INVITE123" }` и актуальными Telegram init data в заголовке `X-Telegram-Init-Data`.
2. Код пригласившего валидируется, создаётся запись в таблице `Referral`.
3. Повторное использование кода или попытка «самоприглашения» завершится ошибкой.

## Начисление награды

- Размер бонуса настраивается в `/admin/economy` (поле «Награда за друга»). Значение хранится в `system_setting` под ключом `referral:reward`.
- После выполнения целевого действия приглашённого вызовите `await completeReferral(inviteeId, metadata?)` из `lib/services/referral`. Функция:
  - Помечает реферала как завершённого и фиксирует `completedAt`.
  - Начисляет звёзды на баланс пригласившего, создаёт запись в `Transaction` и событие безопасности.

## Статистика

`getReferralStats(userId)` возвращает код, число приглашённых, завершённых и ожидающих, а также данные о пригласившем пользователя. Маршрут `GET /api/mini-app/referral` отдаёт те же сведения в JSON.
