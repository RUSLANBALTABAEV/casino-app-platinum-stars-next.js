# Магазин NFT (Telegram Gifts) — техническое задание

## 1. Цель магазина
Полноценный магазин для покупки и продажи NFT‑подарков (Telegram Gifts) только за звёзды. Магазин работает с реальным аккаунтом Telegram для автоматической выдачи подарков после подтверждения через админ‑панель. Пополнение баланса через NFT также происходит автоматически.

## 2. Режим работы магазина

### 2.1. Реальные Telegram Gifts с автоматической выдачей
- **Покупка/продажа**: только за звёзды
- **Выдача NFT**: автоматическая после подтверждения админом через админ‑панель
- **Пополнение через NFT**: автоматическое (userbot фиксирует входящие подарки)
- **Комиссия на выдачу**: 25 звёзд (списывается при создании заявки на выдачу)

### 2.2. Процесс работы
1. Пользователь покупает NFT за звёзды → NFT добавляется в инвентарь (`UserNftGift` со статусом `OWNED`)
2. Пользователь запрашивает выдачу NFT → списывается комиссия 25★ → создаётся заявка → статус меняется на `PENDING_SEND`
3. Админ подтверждает выдачу в админ‑панели → автоматическая отправка через реальный аккаунт Telegram → статус меняется на `SENT`
4. Пользователь отправляет NFT на аккаунт‑кошелёк → userbot фиксирует входящий gift → автоматическое начисление звёзд на баланс

## 3. Что хранится в базе (рекомендуемая модель)

### 3.1. Справочник доступных NFT
Таблица `NftGift`:
- `id`, `name`, `rarity`, `imageUrl`
- `telegramGiftId` (идентификатор gift в Telegram)
- `priceStars`
- `isActive`

### 3.2. Инвентарь пользователя
Таблица `UserNftGift`:
- `userId`, `giftId`, `source`
- `status`: `OWNED`, `PENDING_SEND`, `SENT`, `STAKED`, `CRAFTED`, `SOLD`
- `metadata` (например игра/батл/крафт/заявка на выдачу)

## 4. Как выглядит процесс покупки (общая схема)

1) Пользователь открывает магазин.
2) Получает список активных NFT и цен.
3) Выбирает NFT → отправляет запрос покупки.
4) Сервер проверяет баланс и списывает стоимость.
5) NFT добавляется в инвентарь.
6) Возвращается обновленный баланс + купленный NFT.

## 5. Выдача NFT (отправка через реальный аккаунт Telegram)

### 5.1. Сценарий выдачи (автоматическая после подтверждения админом)
1. Пользователь выбирает NFT из инвентаря со статусом `OWNED` и запрашивает выдачу.
2. Система проверяет баланс и списывает комиссию **25 звёзд**.
3. Создаётся заявка на выдачу (`NftShopOrder` со статусом `PENDING` и типом `BUY`).
4. Статус NFT меняется на `PENDING_SEND`.
5. **Админ подтверждает выдачу в админ‑панели** → статус заявки меняется на `APPROVED`.
6. **Автоматическая отправка через реальный аккаунт Telegram** (userbot отправляет gift пользователю).
7. После успешной отправки статус NFT меняется на `SENT`, заявка → `FULFILLED`.

**Важно**: Выдача происходит автоматически после подтверждения админом, без дополнительных действий со стороны пользователя.

### 5.2. Сценарий пополнения через NFT (автоматический)
1. Пользователь передаёт NFT (Telegram Gift) на аккаунт‑кошелёк (специальный Telegram аккаунт бота).
2. **Userbot автоматически фиксирует входящий gift** через Telegram API.
3. Сервер определяет тип и стоимость gift, конвертирует в звёзды.
4. **Автоматическое начисление звёзд на баланс пользователя**.
5. В историю транзакций записывается пополнение типа `DEPOSIT` с метаданными о полученном NFT.
6. NFT может быть добавлен в инвентарь или сразу конвертирован в звёзды (в зависимости от настроек).

## 6. Покупка и продажа NFT

### 6.1. Покупка NFT
- Покупка работает **только за звёзды**.
- Цена берётся из `NftGift.priceStars`.
- После покупки NFT добавляется в инвентарь пользователя со статусом `OWNED`.
- Баланс пользователя уменьшается на стоимость NFT.

### 6.2. Продажа NFT
- Пользователь может продать NFT из своего инвентаря обратно в магазин.
- Продажа работает **только за звёзды**.
- Цена продажи может быть фиксированной или рассчитываться по формуле (например, 80% от цены покупки).
- После продажи NFT удаляется из инвентаря пользователя (или меняет статус на `SOLD`).
- Баланс пользователя увеличивается на стоимость продажи.

## 7. Дополнительные опции магазина
- Ограничение по остаткам (склад).
- Ограничение по времени (акции).
- Редкие лимитированные подарки.

## 8. Технические требования

### 8.1. Реальный аккаунт Telegram
- Магазин работает с **реальным аккаунтом Telegram** для автоматической выдачи подарков.
- Требуется настройка userbot для отправки и получения Telegram Gifts.
- Аккаунт‑кошелёк для приёма NFT от пользователей.

### 8.2. Автоматизация процессов
- ✅ **Автоматическая выдача** после подтверждения админом
- ✅ **Автоматическое пополнение** при получении NFT на аккаунт‑кошелёк
- ✅ Автоматическое списание комиссии при создании заявки на выдачу

### 8.3. Комиссия на выдачу
- Фиксированная комиссия: **25 звёзд**
- Списывается при создании заявки на выдачу NFT
- Комиссия не возвращается, даже если заявка отклонена админом

### 8.4. Валюта
- Покупка/продажа: **только звёзды** (Telegram Stars)
- Другие валюты не поддерживаются

## 9. Структура данных (уточнение)

### 9.1. Таблица `NftShopOrder` (заявки на выдачу/покупку)
- `type`: `BUY` (покупка) или `SELL` (продажа)
- `status`: `PENDING` → `APPROVED` → `FULFILLED` или `DECLINED`
- `priceStars`: стоимость NFT
- `feeStars`: комиссия (25★ для выдачи)
- `processedById`: ID админа, подтвердившего заявку
- `approvedAt`, `fulfilledAt`: временные метки

### 9.2. Таблица `NftInventoryItem` (склад)
- `status`: `IN_STOCK`, `RESERVED`, `SENT`
- Используется для отслеживания реальных NFT на складе
- Связывается с заявками через `assignedItemId`
